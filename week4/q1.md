# Web Application Security Vulnerability Checklist

| Vulnerability Category      | Description                                                                            | Severity | Basic Countermeasure                                              |
|:---------------------------:|----------------------------------------------------------------------------------------|:--------:|-------------------------------------------------------------------|
| **Injection**               | Unfiltered user data is sent to an interpreter (SQL, LDAP, OS command)                 | High     | Use Parameterized Queries                                         |
| Broken Authentication       | Poor implementation of session management and login logic.                             | High     | Implement Multi-Factor Authentication (MFA)                       |
| Sensitive Data Exposure     | Inadequate protection of data like passwords                                           | High     | Use TLS/SSL (HTTPS) and strong hashing                            |
| Cross-Site Scripting (XXS)  | Malicious scripts are injected into trusted websites and executed in a user's browser. | Medium   | Sanitize inputs and use Content Security Policy                   |
| Broken Access Control       | Users can access data or functions outside of their intended permissions.              | High     | Implement "Deny by Default" and Role-Based Access Control (RBAC). |
